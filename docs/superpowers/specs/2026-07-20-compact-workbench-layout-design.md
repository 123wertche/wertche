# 紧凑型双栏工作台设计

## 目标

在不改变采集、转写、飞书同步和一键流水线行为的前提下，重新组织本地工作台的布局与排版，使主要操作在普通桌面窗口中更集中、更易扫读，并保持手机端可用。

## 页面结构

- 顶部使用紧凑标题栏：产品名称、仅本机提示、服务状态和刷新按钮位于同一视觉区域。
- 主体在宽度不小于 880px 时使用双栏：左栏约 58%，右栏约 42%。
- 左栏依次包含“选择博主”和“运行设置”。
- 右栏包含“一键执行”和“进度与结果”，运行状态始终在同一位置查看。
- 高级工具与危险操作移至主工作区下方，并保持折叠。
- 小于 880px 时自动切换单栏；小于 560px 时操作按钮和表单控件占满可用宽度。

## 博主区域

- 添加链接输入框和添加按钮保持同一行，窄屏改为上下排列。
- 平台筛选、全选与取消选择合并为一条紧凑工具栏。
- 博主列表使用单行选择项；展示选择框、平台标签和可读名称，主页链接作为较弱的第二行信息。
- 列表保持有限高度并内部滚动，避免博主数量增加后把执行区域推到页面底部。
- 已选数量显示在区域标题右侧。

## 运行与结果

- 视频数量、模型和设备使用三列紧凑表单；转写与评论开关紧随其后。
- 页面只保留一个高强调主按钮“开始一键流程”，其他操作均使用次级样式。
- 右栏显示当前阶段、状态文字、进度条和日志；日志区域设置稳定高度，减少运行时布局跳动。
- dry-run 通过后，确认区直接出现在主按钮下方，不使用弹窗，也不遮挡日志。
- 成功、警告和失败状态同时使用文字与颜色，避免只依赖颜色表达。

## 视觉规范

- 使用系统字体，不加载公网字体或新增前端依赖。
- 采用浅灰蓝页面背景、白色工作区、深色正文和蓝色主操作；成功使用绿色，失败使用红色。
- 使用 4/8px 间距节奏，主体字号不小于 14px，输入控件和按钮高度不小于 40px。
- 边框与阴影保持轻量，减少当前页面中大卡片之间的视觉割裂。
- 交互状态使用 150–200ms 的颜色、边框和透明度过渡；支持 `prefers-reduced-motion`。
- 所有键盘焦点保持明显，表单控件使用显式标签，正文对比度达到 WCAG AA。

## 行为与边界

- 保留现有 API、元素 ID 和主要 JavaScript 数据流，避免影响一键流水线。
- 不修改爬虫、转写、飞书字段、任务确认或本地文件处理逻辑。
- 不新增小红书入口，不开放删除操作。
- 页面继续只服务于 `127.0.0.1`。

## 验证

- 运行现有工作台和流水线相关测试。
- 检查桌面双栏布局、375px 手机单栏布局以及无横向滚动。
- 验证四个博主能正常显示和选择，默认视频数仍为 2，设备仍为 GPU 优先自动回退。
- 验证添加博主、平台筛选、全选/取消、启动流水线和确认区域的交互入口仍存在。
- 通过浏览器检查标题层级、按钮标签、焦点状态和页面可见文本。
