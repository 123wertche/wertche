# 一键自动化采集工作台设计

## 目标

将现有本地工作台简化为以“选择博主并运行完整流程”为核心的单页向导。页面同时支持抖音和 B站博主管理，允许单选、多选、全选和随时新增博主，并尽可能自动完成环境准备、候选发现、下载、转写、飞书同步、指标与封面补齐、评论同步、文档发布、验证和导出。

## 范围与安全边界

- 服务仅监听 `127.0.0.1:8765`，不开放公网，不增加账号或多人系统。
- 继续使用项目 `.venv`、现有 Python/Node 脚本和原生 HTML/CSS/JavaScript，不批量升级依赖，不修改全局环境。
- 只支持抖音、B站和飞书，不展示或执行小红书功能。
- 一键操作可以自动执行本地读取、环境准备、dry-run、下载和转写；真正写入飞书前必须统一暂停一次，并要求输入本次 dry-run 生成的确认词。
- 删除飞书记录、删除文档和清理本地文件不进入一键流水线，继续保持范围预览和独立确认；没有可靠限定范围的删除器时拒绝执行。
- 不输出或返回 `feishu-base-config.json`、环境变量或授权信息中的密钥。
- 不覆盖现有用户文件；导出和运行报告使用时间戳路径。重复运行按“平台 + 平台视频 ID”幂等处理。

## 页面结构

页面顶部显示本地服务、项目 Python、Node、lark-cli、ffmpeg、yt-dlp、whisper、飞书授权、项目 Chrome 和 CDP bridge 状态，并提供“一键准备环境”。环境检查结果缓存 30 秒；任务、流水线状态和增量日志每 2 秒刷新，避免频繁启动工具检查进程。

主操作区为统一博主列表：

- 按抖音/B站筛选和搜索。
- 支持全选、单选和多选。
- 支持粘贴一个或多个主页链接随时新增。
- 抖音以 `sec_uid` 去重，B站以 `MID` 去重。
- 无法可靠识别的平台或身份明确报错，不创建猜测数据。

主按钮为“运行所选博主完整流程”。默认设置显示在按钮附近：每位博主 2 条最新非置顶视频、设备自动、转写开启、评论同步开启。模型、设备、每博主数量和阶段开关放在折叠的高级设置中。原有独立脚本按钮也移入“高级操作”，用于诊断和单阶段重试。

页面下部显示总任务阶段、当前博主/视频、实时日志、成功与失败记录、飞书链接、manifest、导出路径和最终验证报告。危险操作继续单独折叠。

## 统一博主管理

新增本地统一博主清单，保存稳定的本地 ID、平台、主页 URL、平台身份 ID、显示名称、启用状态和来源。已有 `douyin-creators.json` 数据在首次读取时合并展示，不覆盖原文件；B站现有博主从飞书读取并与本地新增项按 MID 合并。

新增抖音主页时解析 `sec_uid`。新增 B站主页时解析 MID；短链或无法直接得到 MID 的链接只允许通过受控解析流程转换，转换失败时标记“无法识别”。同一平台身份不重复保存。

B站本地新增博主如果飞书博主表中不存在，可用于候选发现和本地 dry-run；正式视频同步前，流水线将“预计创建的 B站博主记录”列入统一飞书写入预览。只有用户输入确认词后才创建博主记录并取得动态 record ID，禁止硬编码 record ID。

## 一键启动

项目根目录新增 `启动工作台.cmd`，双击后：

1. 检查 `127.0.0.1:8765` 是否已有本项目服务。
2. 未运行时使用 `.venv\Scripts\python.exe` 隐藏启动 `local_workbench.py`。
3. 轮询轻量 `/api/ready`，不调用耗时环境检查。
4. 就绪后使用系统默认浏览器打开 `http://127.0.0.1:8765/`。
5. 已运行时直接打开页面，不重复创建服务。

服务记录自己启动的服务、项目 Chrome 和 bridge PID。页面可停止这些本项目进程，但不终止普通 Chrome 或其他浏览器。启动失败时命令窗口保留简短错误和日志文件地址，不显示密钥。

## 一键流水线

后端增加总任务编排器。每次运行使用选中博主、平台、数量、模型、设备和阶段开关的规范化快照作为任务指纹。

流水线阶段固定为：

1. 轻量服务就绪检查。
2. 环境工具和飞书授权预检。
3. 启动或复用项目专用 Chrome，启动或复用 CDP bridge，并检查 9333/3457。
4. 对所选抖音和 B站博主执行候选 dry-run，默认每位博主 2 条最新非置顶视频。
5. 汇总候选、已存在视频、预计新增/更新视频、预计新增 B站博主、预计创建文档和数据不可用项。
6. 生成一个与任务指纹绑定、30 分钟有效的确认词，并暂停总任务。
7. 用户输入确认词后，验证参数未变化，继续正式流程。
8. 下载视频并保留原文件。
9. 后处理和转写。
10. 按平台视频 ID 同步飞书视频记录。
11. 补摘要、路径、处理状态、官方指标和封面附件。
12. 同步 B站评论。
13. 发布转写文档并回写链接。
14. 重新读取飞书验证记录、指标、封面 token、文档链接和幂等结果。
15. 导出视频表 JSON/XLSX、建立整理副本并生成运行报告。

用户未确认时，总任务保持 `awaiting_confirmation`，不会自动超时后写入。确认词过期或参数变化时重新执行相关 dry-run。单个博主或视频失败不终止其他项；最终状态可为 `succeeded`、`partially_succeeded` 或 `failed`。

## GPU 优先与 CPU 降级

默认设备为 `auto`。预检先检测 NVIDIA 驱动、CUDA 和当前转写后端的实际可用性，并进行轻量初始化测试。检测成功后使用 GPU；检测失败直接选择 CPU。

GPU 转写阶段出现初始化失败、显存不足或后端运行异常时，只重试当前转写阶段并自动降级 CPU，不重新下载视频、不重复创建飞书记录。每个视频记录实际使用设备和降级原因。高级设置允许明确选择 GPU 或 CPU；明确选择 GPU 时仍会在不可用时暂停并提示，而不是静默伪装为 GPU 成功。

## API 与状态

新增或调整以下边界：

- `GET /api/ready`：仅返回服务就绪、版本和监听地址，不运行外部工具。
- `GET /api/health?force=0|1`：返回缓存的环境状态；`force=1` 执行强制刷新。
- `GET /api/creators?platform=...`：返回统一博主列表和选择所需的非敏感字段。
- `POST /api/creators/preview`：解析、去重并预览新增链接。
- `PUT /api/creators`：原子保存通过预览的本地博主清单；飞书创建仍由流水线确认阶段处理。
- `POST /api/pipelines`：创建一个总任务，只接受白名单结构化参数。
- `GET /api/pipelines`、`GET /api/pipelines/<id>`、`GET /api/pipelines/<id>/log?offset=N`：返回总任务、阶段、子任务和增量日志。
- `POST /api/pipelines/<id>/confirm`：校验确认词和任务指纹后继续写入阶段。
- `POST /api/pipelines/<id>/retry`：只重试明确失败且可安全重试的阶段。
- `POST /api/project/start`、`POST /api/project/stop`：管理服务拥有的项目 Chrome/bridge；停止服务本身由启动器或独立受控端点处理。

HTTP 服务不接受原始命令、shell、任意脚本路径、项目外文件路径、固定视频 ID 或固定飞书 record ID。

## 错误与恢复

- 页面刷新或关闭不会终止总任务。总任务状态和非敏感摘要写入 `runtime/workbench/tasks`，服务重启后把未完成任务恢复为可检查状态；不会盲目重启飞书写入。
- 页面为空、媒体暂时缺失或抖音详情响应未捕获时沿用有限重试、退避和重开标签页策略。
- GPU 降级、工具缺失、飞书授权失效、确认词过期、参数变化、CDP 不可达和脚本退出码均显示明确阶段和建议动作。
- 没有数据时显示“数据不可用”；未完成重新读取验证的写入显示“未验证”。
- 单阶段重试复用同一 manifest 和动态 ID 映射，先检查现有记录和附件，避免重复记录、封面或文档。

## 测试与验收

- 单元测试：统一博主链接解析、平台识别、sec_uid/MID 去重、原子保存、选择快照、默认数量 2、动作白名单、任务指纹、确认词有效期、GPU 检测和 CPU 降级。
- 编排测试：阶段顺序、dry-run 暂停、确认后续跑、参数变化拒绝、失败隔离、部分成功、服务重启恢复和安全重试。
- API 测试：轻量 ready、健康缓存/强制刷新、博主预览与保存、总任务创建/确认/日志，以及原始命令和项目外路径拒绝。
- 启动器测试：首次启动、重复双击、端口已占用但不是本项目服务、服务启动失败和自动打开页面。
- 集成测试：选择单个和多个抖音/B站博主，验证默认每位 2 条候选；真实写入仍由用户在页面明确确认。
- 验证测试：重新读取飞书，检查五项基础指标为数字、三项留存指标为数值或“数据不可用”、封面 token、文档链接和平台视频 ID 幂等。
- 页面测试：简化向导、博主筛选/多选/新增、高级操作折叠、实时阶段、错误恢复、320px 和桌面布局。
- 完整回归：运行全部现有测试、安全扫描和实际本地页面渲染；最终关闭测试启动的服务、Chrome 和 bridge，不影响用户其他浏览器。
